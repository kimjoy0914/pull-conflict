# 2026년 3월8일 충돌사례1번 실습
상황 설명
웹레포에 커밋아이디가 있고 로컬 레포에서 수정이 발생한 뒤
git add .하고 git commit -m "메시지" 한 경우에 대한 상황임.
즉 원격레포 커밋아이디 보다 로컬레포 커밋아이디가 한 개 더 있는 상황임.
이 경우는 충돌이 아니라 git pull origin main하면 되는 상황이지 충돌이 아님.

다시 말해 
main 로컬 레포와 origin/main 원격레포가 분리되어진 상태임.
원격레포 보다 main로컬 레포가 하나 위에 있는 상황인 것임.
